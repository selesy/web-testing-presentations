Selenium Overview

Selenium 1

  • Javascript "shim"
  • Inserted into frame after page load
  • Couldn't detect DOM changes
  • Selenium IDE provided a "test recorder"

Selenium 2

  • Browser plugin
  • Simulates user behaviors
  • Provides tools to manage DOM changes
  • No IDE or test recorder

Selenium Grid

  • Allows remote cluster to execute tests
  • Mix and match OS, browsers and versions
  • The same test can be run on multiple profiles
  • Commercial support - you don't have to manage your own infrastructure

Get Screenshot

  • It's not just about application behavior
  • Visual "Look & Feel" is also important
  • Helps find those "off-by-one" pixel errors
  • Not supported on every browser
